Spring constraint diagram toy

A small companion widget for diagrams that argue back. Enter nodes and preferred edge lengths, then let the system pull, push, damp, and settle into a compromise.

Drag any node on the canvas. Fixed nodes stay pinned; free nodes keep negotiating with their constraints.

Inputs

Use plain JSON arrays. Reset reloads the data and restarts the search.

Nodes

Required: id, x, y. Optional: fixed, mass.

Edges and constraints

Required: source, target, length. Optional: stiffness.

Simulation controls

Spring model: F = k(L - L0). Gravity, when enabled, adds Fy = mg straight down for each free node, but only if at least one node is fixed. Motion model: a = F / m, then v = (1 - damping)(v + a dt), and x = x + v dt.

Results

Blue edges are stretched; red-orange edges are compressed.
